Blindfold automatically detects and protects 60+ pre-trained types of personally identifiable information (PII) across multiple languages. Our detection engine uses flexible, natural language entity names rather than strict uppercase labels. Additionally, you can define custom entity types specific to your use case using zero-shot detection.

## Entity Detection Confidence

All detected entities include a confidence score:

| Score Range | Confidence Level | Action |
|-------------|------------------|--------|
| 0.95 - 1.0  | Very High        | Automatically protected |
| 0.85 - 0.94 | High             | Automatically protected |
| 0.70 - 0.84 | Medium           | Protected by default |
| 0.50 - 0.69 | Low              | Requires threshold adjustment |
| < 0.50      | Very Low         | Not detected by default |

### Adjusting Detection Threshold

You can adjust the confidence threshold:

```python
# Only detect high-confidence entities
result = client.tokenize(
    text="Sample text with PII",
    score_threshold=0.85  # Only entities with score >= 0.85
)
```

## Entity Filtering

Process only specific entity types using natural language entity names:

```python
# Only protect emails and phone numbers
result = client.tokenize(
    "John Doe (john@example.com) called from +1-555-1234",
    entities=["email address", "phone number"]
)
# Output: "John Doe (<email_address_1>) called from <phone_number_1>"
# Note: "John Doe" is NOT tokenized because "person" was not included

# Mix entity types
result = client.tokenize(
    "Patient: Sarah Jenkins, DOB: 1985-04-12, Medication: Metformin 500mg",
    entities=["person", "date of birth", "medication"]
)
# Output: "Patient: <person_1>, DOB: <date_of_birth_1>, Medication: <medication_1>"

# Financial data only
result = client.tokenize(
    "Card: 4532-7562-9102-3456, CVV: 123, Expires: 12/25",
    entities=["credit card number", "cvv", "credit card expiration date"]
)
# Output: "Card: <credit_card_number_1>, CVV: <cvv_1>, Expires: <credit_card_expiration_date_1>"
```

## Custom Entity Types

Blindfold supports **custom entity detection** beyond the 60+ pre-trained types. Define your own entity types specific to your industry or use case using natural language descriptions.

<Note>
  Our detection engine uses zero-shot learning, allowing you to specify custom entity labels without any training or configuration. Just describe what you want to detect in plain English!
</Note>

```python
# Detect custom entities specific to your domain
result = client.tokenize(
    "Booking ref: BK-2024-XYZ, Work ID: WRK-456",
    entities=["booking reference", "work identifier", "internal code"]
)

# Industry-specific entities
result = client.tokenize(
    "Patient admitted to Ward 5B, assigned Doctor ID: DOC-789",
    entities=["ward number", "doctor identifier"]
)

# E-commerce entities
result = client.tokenize(
    "Order #12345, SKU: PROD-ABC-789, Tracking: 1Z999AA10123456784",
    entities=["order number", "product sku", "tracking number"]
)
```

### How Custom Entities Work

1. **Flexible Labels**: Describe entities in natural language (e.g., "employee ID", "project code", "customer reference")
2. **No Training Required**: The model understands entity types on-the-fly using zero-shot detection
3. **Combine with Pre-trained**: Mix custom entities with standard PII types in the same request
4. **Industry Adaptable**: Works across healthcare, finance, legal, HR, and custom domains

```python
# Mix standard and custom entities
result = client.tokenize(
    "Customer John Doe (ID: CUST-123) purchased item SKU-789 on 2024-01-15",
    entities=["person", "email address", "customer id", "product sku", "date of birth"]
)
# Detects: person name, any email (if present), customer ID, product SKU, and dates

# Healthcare example mixing standard and custom
result = client.tokenize(
    "Patient: Mary Smith, DOB: 1980-05-15, Ward: 3A, Attending: Dr. Johnson (ID: DOC-456)",
    entities=["person", "date of birth", "ward number", "doctor identifier"]
)
```

<Tip>
  Custom entity detection is particularly useful for industry-specific identifiers, internal codes, or proprietary data formats that aren't covered by the 60+ standard entity types.
</Tip>
