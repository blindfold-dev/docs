Blindfold automatically detects and protects over 50 types of personally identifiable information (PII) across multiple languages and regions.

## Core Entities

<AccordionGroup>
  <Accordion title="PERSON" icon="user">
    **Description**: A person's full name.

    **Examples**:
    - John Doe
    - Jane Smith
    - Dr. Michael Johnson

    **Detection Score**: High confidence (0.85+)
  </Accordion>

  <Accordion title="EMAIL_ADDRESS" icon="envelope">
    **Description**: An email address.

    **Examples**:
    - john@example.com
    - user.name+tag@domain.co.uk
    - admin@company.org

    **Detection Score**: Very high confidence (0.95+)
  </Accordion>

  <Accordion title="PHONE_NUMBER" icon="phone">
    **Description**: A phone number.

    **Examples**:
    - +1-555-123-4567
    - (555) 123-4567
    - +44 20 7946 0958

    **Detection Score**: High confidence (0.80+)
  </Accordion>

  <Accordion title="CREDIT_CARD" icon="credit-card">
    **Description**: A credit card number.

    **Examples**:
    - 4532-7562-9102-3456 (Visa)
    - 5425-2334-3010-9903 (Mastercard)
    - 3782-822463-10005 (Amex)

    **Detection Score**: Very high confidence (0.95+)

    <Warning>
      Credit card detection uses Luhn algorithm validation.
    </Warning>
  </Accordion>

  <Accordion title="IP_ADDRESS" icon="network-wired">
    **Description**: An IPv4 or IPv6 address.

    **Examples**:
    - 192.168.1.1 (IPv4)
    - 2001:0db8:85a3:0000:0000:8a2e:0370:7334 (IPv6)
    - 10.0.0.1 (Private IPv4)

    **Detection Score**: Very high confidence (0.99+)
  </Accordion>

  <Accordion title="LOCATION" icon="location-dot">
    **Description**: A physical location such as city, state, or address.

    **Examples**:
    - New York, NY
    - 123 Main Street, Boston, MA
    - Paris, France

    **Detection Score**: Medium to high confidence (0.70+)
  </Accordion>

  <Accordion title="ORGANIZATION" icon="building">
    **Description**: A company or organization name.

    **Examples**:
    - Microsoft Corporation
    - Harvard University
    - World Health Organization

    **Detection Score**: Medium confidence (0.70+)
  </Accordion>
</AccordionGroup>

## Custom Entity Types

You can also define custom entity types for your specific use case:

```python
from blindfold import Blindfold

client = Blindfold(api_key="your-api-key")

# Detect specific custom patterns
result = client.tokenize(
    "Employee ID: EMP-12345",
    entities=["PERSON", "EMAIL_ADDRESS"],
    custom_patterns=[
        {
            "name": "EMPLOYEE_ID",
            "pattern": r"EMP-\d{5}",
            "score": 0.9
        }
    ]
)
```

## Entity Detection Confidence

All detected entities include a confidence score:

| Score Range | Confidence Level | Action |
|-------------|------------------|--------|
| 0.95 - 1.0  | Very High        | Automatically protected |
| 0.85 - 0.94 | High             | Automatically protected |
| 0.70 - 0.84 | Medium           | Protected by default |
| 0.50 - 0.69 | Low              | Requires threshold adjustment |
| < 0.50      | Very Low         | Not detected by default |

### Adjusting Detection Threshold

You can adjust the confidence threshold:

```python
# Only detect high-confidence entities
result = client.tokenize(
    text="Sample text with PII",
    score_threshold=0.85  # Only entities with score >= 0.85
)
```

## Entity Filtering

Process only specific entity types:

```python
# Only protect emails and phone numbers
result = client.tokenize(
    "John Doe (john@example.com) called from +1-555-1234",
    entities=["EMAIL_ADDRESS", "PHONE_NUMBER"]
)
# Output: "John Doe (<EMAIL_ADDRESS_1>) called from <PHONE_NUMBER_1>"
# Note: "John Doe" is NOT tokenized
```
